# Document Biological Coordination Model

## Why

Colony already has a durable biological-coordination contract, but contributors need a short user-facing explanation that keeps the model practical. The docs should explain why agents pull work, why signals decay, and why Queen publishes structure without sounding like a gimmick.

## What Changes

- Add a concise biological coordination model section to `README.md`.
- Link the README to the durable OpenSpec contract.
- Add a Queen-specific practical note to `docs/QUEEN.md`.
- Include examples for stale claims, proposals, ordered waves, and local context before editing.

## Impact

Docs become clearer for contributors without changing runtime behavior. OpenSpec remains the source of truth for behavior contracts.
